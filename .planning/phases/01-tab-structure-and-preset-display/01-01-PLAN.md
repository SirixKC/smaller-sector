Completely rewrite `data/config/LunaSettings.csv` with the following structure. The CSV has 9 columns: `fieldID,fieldName,fieldType,defaultValue,secondaryValue,fieldDescription,minValue,maxValue,tab`

CRITICAL RULES:
- Every non-blank row MUST have column 9 (tab) populated. No blank rows allowed (remove all `,,,,,,,,` spacer rows). An empty tab column creates an orphaned "General" tab.
- All 17 existing field IDs must be preserved EXACTLY: `preset`, `cruiserToFrigate`, `cruiserToDestroyer`, `capitalToFrigate`, `capitalToDestroyer`, `capitalToCruiser`, `cruiserCrewMult`, `cruiserSupplyMult`, `cruiserFuelMult`, `capitalCrewMult`, `capitalSupplyMult`, `capitalFuelMult`, `cruiserBuildCostMult`, `capitalBuildCostMult`, `cruiserDmodCount`, `capitalDmodCount`, `factionBlacklist`. Changing any ID breaks saved user settings.
- Presets tab rows MUST come first in the CSV (first unique tab name = default tab).
- Use `%%` for literal percent signs in Text field values (LunaLib CSV parsing interprets single `%`).
- New field IDs for Text/Header/Radio display elements use `ss_` prefix to avoid any future collision.
- Text fields use `[brackets]` for highlighted text.
