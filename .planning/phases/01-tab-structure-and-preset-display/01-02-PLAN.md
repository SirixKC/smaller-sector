<objective>
Add "load preset values into sliders" functionality to PresetListener so users on the Custom preset can copy a named preset's values as a starting point, then customize from there.

Purpose: Without this, Custom users have no way to quickly populate sliders with a preset's values as a baseline. They would need to manually type every value. The load-preset radio on the Configuration tab triggers PresetListener to copy values and reset itself.

Output: Updated PresetListener.java with load-preset handling.
</objective>

   - Reset the `ss_load_preset` radio back to `"-- None --"`:
     a. Update via JSON file: load the CommonDataJSONObject, put `"ss_load_preset"` = `"-- None --"`, save.
     b. Update via `LunaSettingsLoader.INSTANCE.loadSettings(MOD_ID, true)` to reload in-memory.
     c. Update the live UI radio element using the VERIFIED API (confirmed from LunaLib source `LunaUIRadioButton.kt`):
        ```java
        // In addedElements iteration, after finding ss_load_preset:
        if (element instanceof LunaUIRadioButton) {
            LunaUIRadioButton radio = (LunaUIRadioButton) element;
            radio.setValue("-- None --");
            for (LunaUIButton button : radio.getButtons()) {
                if ("-- None --".equals(button.getButtonText().getText())) {
                    button.setSelected();
                    break;
                }
            }
        }
        ```
        `LunaUIRadioButton` extends `LunaUIBaseElement` (appears in `addedElements`). Its `value` is a public `var String?` (Java: `getValue()`/`setValue()`).
