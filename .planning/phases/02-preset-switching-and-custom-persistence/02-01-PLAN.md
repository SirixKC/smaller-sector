<task type="auto">
  <name>Task 2: Add backup and restore logic to PresetListener</name>
  <files>src/smallersector/PresetListener.java</files>
  <action>
  Add custom value backup and restore functionality to `PresetListener.java`. Use the research code examples as reference but integrate cleanly into the existing code structure.

  **Constants to add (at class level):**
  - `private static final String BACKUP_FILE = "SmallerSectorCustomBackup.json";`
  - `private static final String[] BACKUP_SETTINGS_INT` array containing: `cruiserToFrigate`, `cruiserToDestroyer`, `capitalToFrigate`, `capitalToDestroyer`, `capitalToCruiser`, `cruiserDmodCount`, `capitalDmodCount`
  - `private static final String[] BACKUP_SETTINGS_DOUBLE` array containing: `cruiserCrewMult`, `cruiserSupplyMult`, `cruiserFuelMult`, `capitalCrewMult`, `capitalSupplyMult`, `capitalFuelMult`, `cruiserBuildCostMult`, `capitalBuildCostMult`

  Note: `factionBlacklist` is excluded from backup -- per user decision, consistent with Phase 1's exclusion from preset loading.

  **New method `backupCustomValues()`:**
  - Uses `JSONUtils.loadCommonJSON(BACKUP_FILE)` to load/create the backup file
  - Reads each setting from LunaSettings via `LunaSettings.getInt/getDouble(MOD_ID, key)` using the two arrays above
  - Puts non-null values into the backup JSON object
  - Calls `backup.save()`
  - Wraps in try-catch, logs warning on failure

  **New method `loadBackupValues()` returning `Map<String, Object>`:**
  - Uses `JSONUtils.loadCommonJSON(BACKUP_FILE)` to load the backup file
  - If backup is null or `backup.length() == 0`, returns empty map (no backup exists)
  - Iterates `BACKUP_SETTINGS_INT` and tries `backup.getInt(key)` for each, catching exceptions per-key (handles new settings not in backup -- per user decision, those settings keep their current values)
  - Iterates `BACKUP_SETTINGS_DOUBLE` and tries `backup.getDouble(key)` for each, same per-key try-catch
  - Returns the populated map
  - Wraps outer logic in try-catch, logs warning on failure

  **New method `hasBackup()` returning boolean:**
  - Uses `JSONUtils.loadCommonJSON(BACKUP_FILE)` and returns `backup != null && backup.length() > 0`
  - Returns false on any exception

  **Modify `settingsChanged()` method:**
  - After the existing `handleLoadPreset(currentPreset)` call, add a call to `backupCustomValues()` BUT only when the active preset is Custom:
    ```java
    // Backup custom values when saving with Custom preset active
    if ("Custom".equals(currentPreset)) {
        backupCustomValues();
    }
    ```

  **Modify `handleLoadPreset()` method:**
  - Add a new case `"Restore Custom Values"` to the switch statement, BEFORE the `default` case:
    ```java
    case "Restore Custom Values":
        if ("Custom".equals(currentPreset)) {
            Map<String, Object> restoredValues = loadBackupValues();
            if (!restoredValues.isEmpty()) {
                // Apply restored values using existing pipeline
                updateChangedSettings(restoredValues);
                updateUIElements(restoredValues);
                // Persist to LunaSettings JSON
                CommonDataJSONObject data = JSONUtils.loadCommonJSON(
                    "LunaSettings/" + MOD_ID + ".json",
                    "data/config/LunaSettingsDefault.default"
                );
                if (data != null) {
                    for (Map.Entry<String, Object> entry : restoredValues.entrySet()) {
                        data.put(entry.getKey(), entry.getValue());
                    }
                    data.save();
                }
                LunaSettingsLoader.INSTANCE.loadSettings(MOD_ID, true);
                log.info("Restored custom values from backup");
            } else {
                log.info("No custom values backup found -- restore skipped");
            }
        } else {
            log.info("Restore ignored: active preset is not Custom");
        }
        resetLoadPresetRadio();
        return;
    ```
  - Note: The `resetLoadPresetRadio()` call and `return` are placed INSIDE the case (before break/fall-through), matching the pattern of the existing preset loading cases. The radio always resets regardless of outcome.

  **Add a logger field** if not already present:
  - `private static final org.apache.log4j.Logger log = com.fs.starfarer.api.Global.getLogger(PresetListener.class);`

  **Do NOT:**
  - Auto-restore when switching TO Custom (per user decision -- switching to Custom just unlocks editing)
  - Add a confirmation dialog (per user decision -- immediate switch)
  - Backup when a named preset is active (would overwrite custom backup with preset values -- Pitfall 1 from research)
